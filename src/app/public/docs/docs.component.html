API Documentation

/parseIngredients POST

The /parseIngredients endpoint parses a list of raw ingredient strings

Request

Fields

  • ingredients: A list of ingredient strings to parse.
    • This list must be between 1 and 100 elements.
    • Each string in the list must be 1,024 characters or fewer.
    • The strings may be HTML- or URL-encoded. Any HTML tags in the ingredient strings will be ignored.

Response

Fields

  • results: If the request was successful, contains a list of dictionaries with the keys "ingredientRaw", "ingredientParsed", and "error".
  • ingredientRaw: The original user-supplied ingredient string that corresponds with this parse result.
  • ingredientParsed: A dictionary cotaining the ingredient as structured data, containing the keys "quantity", "unit", "product", and "preparationNotes".
  • quantity: A number representing the quantity represented in the ingredient string, or null if no quantity was detected in the ingredient string..
  • unit: A string containing the unit of measure in the ingredient string, or null if no unit of measure was found.
  • product: The original user-supplied ingredient string that corresponds with this parse result.
  • preparationNotes: The original user-supplied ingredient string that corresponds with this parse result.
  • error: An error message that describes what caused a particular ingredient to fail or the entire request to fail.

Known Limitations

Can't parse mixtures of units.
